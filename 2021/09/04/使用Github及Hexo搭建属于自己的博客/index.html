使用Github及Hexo搭建属于自己的博客

觉得看文字麻烦的可以转移至B站（如果可以的话给作者一个三连吧）

关于使用中的一些问题我会在最下面进行总结

一.工具准备

1.介绍

  • Node.js：由Ryan Dahl开发，是一个基于Chrome V8引擎的JavaScript运行环境，它让JavaScript成为与PHP、Python、Perl、Ruby等服务端语言平起平坐的脚本语言。
  • Git：是一个开源的分布式版本控制系统,用于敏捷高效地处理任何或小或大的项目。

这两个可以从百度上下载到最新版，不想看英文的可以看谷歌翻译，上面的B站视频里面也有链接（本人就是从那里下载的，安装的时候全点的下一步，不过那个可能版本有点低导致产生问题）。

  • Hexo：基于Node.js的博客框架，可以生成博客页面。
  • Github：GitHub是一个面向开源及私有软件项目的托管平台，Github的仓库（repository）在博客创建的过程中存储博客的内容，承担服务器的作用。

2.安装测试

按windows（键盘左下角的那个矩形）+R，输入cmd。

出现了一个黑框框。

输入node -v和npm -v和git –version，出现相应的版本号则成功。

在npm注册表中有hexo-cli一项，输入npm install hexo-cli -g安装。

出现+ hexo-cli@版本号则安装完成，然后输入hexo -v检验是否安装成功。

二.搭建服务器（Github仓库）

1.创建Github账号（链接）

点击右上角的sign up，一步步来就行了。

一定要记住你的注册名，你的邮箱和你的密码

2.建立一个仓库

点开这个

仓库的名称格式：注册名.github.io

这是一个特定的仓库名，一定要准确。说明可写可不写，下面选择Public和Add a README file，点击创建

三.关联与绑定（给Github绑定一个SSH秘钥）

首先进入你电脑中的任何一个文件夹，在空白处右键鼠标，点击Git Bash Here，输入ssh，检查是否装过ssh（安装Git的时候默认带了这个）

输入ssh-keygen -t rsa -C “你的Github绑定邮箱地址”，连敲四次回车（每次都要等显示完全再敲），敲完不要关闭呀。

然后在C盘中找到生成的文件，复制id_rsa.pub中的密码（注意是.pub文件,因为这是公钥）

然后打开Github，点击头像，找到Settings一项，在左边找到SSH and GPG keys，点击右上的New SSH key

Title名字无所谓，下面粘贴你刚才复制的公钥密码。然后就这样了。（可能会给你发邮件）

然后回到刚才的控制面板，输入ssh -T git@github.com，回车后输入yes

warning不用管，显示Hi那一行就说明绑定成功了，你存放秘钥的那个文件夹生成一个known_hosts的文件

四.在本地生成一个博客的内容

在任意一个盘新建一个文件夹，打开Git Bash Here，输入hexo init进行初始化，你会发现文件夹里多了很多东西（从Github作者那里下载到的）。

然后输入hexo s静态生成本地的hexo页面（然后你的防火墙可能会给你警告，允许访问就好）

然后进入里面的地址，按Ctrl+C可以关闭你的服务器，这样刷新页面你就看不到这个了。

六.在搭建博客的过程中我遇到的问题

1.在验证npm和hexo的时候无法显示：

如果显示不存在的话，可能是因为你还没安装完就输入这些指令了，或者是你在安装之前就已经打开这个界面，只需关掉重开就好了。

2.在下载hexo的时候出现ERR问题：

原因：输入的指令错误，可以看到，hexo-cli被我误写成了hexo-cil

3.在下载hexo的时候出现WARN问题：

可能是版本过低（B站那个版本有些低了，我卸载重下的最新版）

4.输入hexo d时出现如下错误：

fatal: unable to auto-detect email address (got z@DESKTOP-DPE3A08.(none))
error: src refspec HEAD does not match any
error: failed to push some refs to https://github.com/seekwhale13/seekwhale13.ithub.io.git

可以尝试在控制台中输入这些：
git config –global user.email 你的邮箱
git config –global user.name 你的Github名字

然后再输入hexo d

5.输入hexo d时出现如下错误

这说明你网不好，连接不稳定（毕竟是外网），我刷了1个小时才好。。。
